Reference Analysis

Design brief

These guidelines synthesise two references: a military-inspired apparel catalog (structured, utility-forward) and a minimal warm-toned editorial storefront (typographic, atmosphere-first). The result is a modern fashion e-commerce design system that balances discovery utility with brand personality.

Reference A — utility catalog
Left sidebar filters, dense product grid, functional badges (NEW, ONLINE EXCLUSIVE), structured category taxonomy, high information density.
Reference B — editorial hero
Warm beige/cream palette, oversized display type overlaid on photography, horizontal social links bar, scrolling marquee ticker, maximal whitespace.
Visual Identity

Color system

Neutral foundation — always warm, never cold
Build on warm off-whites and near-blacks. Avoid pure #FFFFFF and #000000 — they read as generic. Use a warm paper white for backgrounds and a charcoal (not black) for type.

Single accent rule
Use one accent color (warm orange-red) sparingly: badges only, sale prices, and interactive hover states. Never use it for decorative fills. This gives it maximum signal power.

Hero rules
Display type (48–72px) overlaps the hero photograph using z-index layering — the model image sits below the text. Use a warm background (#F5F2EF) not pure white. Social links sit below the hero as a horizontal row, all caps, even spacing. CTA uses underline-only style, no filled button.
Components

Product cards

Image area
Square or 3:4 ratio. Warm off-white (#EDEAE6) background. No hard borders — background contrast defines the card edge. 4px corner radius max.
Badges
Position: top-left corner. Only two types: NEW (green fill) and ONLINE EXCLUSIVE (accent fill). Never stack more than one badge. All caps, 10px, 500 weight.
Rating stars
Shown above product name only when reviews exist. 5-star scale using filled/empty dots or stars. Show count in muted text: "71 reviews".
Interaction States

Hover and active states

Do
Subtle background shift on product card hover (secondary bg → primary bg). Use border color transition from tertiary to secondary. Keep transitions under 150ms ease.
Don't
Avoid box-shadows, lift effects, or scale transforms on product cards. They break the flat editorial feel and cause layout jank on dense grids.
Do
Underline-style CTAs (Shop Now, View All) with animated width transition. Arrow icons animate 4px right on hover.
Don't
Avoid filled color buttons for secondary actions. Reserve solid dark fills only for primary checkout/add-to-cart actions.

Responsive Breakpoints

Responsive behavior

Mobile (<768px)
Filters collapse into a slide-up drawer. Product grid: 2 columns. Nav collapses to hamburger. Hero type scales down to 40px.
Tablet (768–1024px)
Sidebar collapses to horizontal filter chips row. Product grid: 2–3 columns. Navigation stays horizontal.
Desktop (1024px+)
Full sidebar visible. 3-column product grid (4 optional on wide viewports). Two-row navigation. Marquee ticker full width.
Design Principles

Core principles

Utility first, personality second
Filters, search, sort controls, and product information must work before any editorial styling is layered on top. Function drives the skeleton; the warm palette and type treatment are the skin.
Photography does the heavy lifting
The UI is intentionally neutral so that product photography reads clearly. Avoid competing background colors, patterns, or decorative elements near product images.
Restraint with accent color
The orange-red accent color should feel urgent and rare. If it appears on more than 5% of the visible page at any given time, dilute its usage further.
No dark mode required
The warm off-white palette is the brand. Avoid implementing a dark mode unless specifically requested — it fundamentally changes the aesthetic character of the site.
